# 软件功能实现路径总览

> 用途：为后续前台重构、功能定位、问题排查和二次开发提供一份“从功能到代码落点”的快速索引。
> 范围：聚焦 EVA 当前软件功能的实现路径，不展开推理算法细节。

---

## 1. 总体执行主链路

EVA 当前整体执行链路可以概括为：

1. `main.cpp` 启动应用、创建主窗口 `Widget`、增殖窗口 `Expend`、工具执行器 `ToolExecutor`、网络客户端 `NetClient`、MCP 管理器 `xMcp`。
2. 主窗口 `Widget` 负责 UI 交互、状态机、会话输入组织、历史恢复、输出渲染、工程环境编排。
3. `SessionController` 负责把 UI 输入整理成发送任务与消息快照。
4. `NetClient` / `xNet` 负责构造 OpenAI 兼容请求、发送 HTTP/SSE 请求并解析流式响应。
5. 当模型返回 tool call 时，`ToolFlowController` 解析工具调用，交由 `ToolExecutor` / `xTool` / `xMcp` 执行。
6. 工具结果再回注到会话消息中，必要时继续发起下一轮请求，形成工具回环。
7. 输出最终由 `Widget` 渲染到主界面，同时历史会话与状态更新到存储层。

核心入口文件：

- 启动入口：`src/main.cpp`
- 主界面：`src/widget/widget.cpp`, `src/widget/widget.h`
- 会话编排：`src/core/session/*`
- 工具回环：`src/core/toolflow/*`
- 网络：`src/service/net/*`, `src/xnet.*`
- 工具执行：`src/service/tools/*`, `src/xtool.*`
- MCP：`src/xmcp.*`
- 历史/存储：`src/storage/*`

## 2. 主窗口聊天功能实现路径

### 2.1 用户发起聊天

主路径：

- UI 与状态入口：`src/widget/widget.cpp`
- 会话发送与流程控制：`src/widget/widget_session.cpp`, `src/widget/widget_state.cpp`, `src/widget/widget_toolflow.cpp`
- 会话编排：`src/core/session/session_controller.*`
- 请求发送：`src/service/net/net_client.*`, `src/xnet.*`
- 输出渲染：`src/widget/widget_output.cpp`, `src/widget/widget_link.cpp`, `src/widget/widget_records.cpp`

典型过程：

1. 用户在主窗口输入消息或附件。
2. `Widget` 收集输入、上下文、当前约定、工具状态、模型状态。
3. `SessionController` 生成 `RequestSnapshot` 与消息数组。
4. `NetClient` 调用 `xNet` 发起请求。
5. `xNet` 解析 SSE，把输出流式回传 UI。
6. `Widget` 实时渲染输出，并在结束时写入历史。

### 2.2 输出渲染

关键文件：

- `src/widget/widget_output.cpp`：主输出区文本流渲染
- `src/widget/widget_records.cpp`：记录条与消息片段展示
- `src/widget/widget_link.cpp`：链接模式相关输出、高亮与部分消息格式化

关注点：

- assistant / tool / think 不同角色的视觉呈现
- 流式输出拼接
- KV、速度、状态信息同步
- 工具结果高亮与结构化显示

## 3. 工具调用闭环实现路径

### 3.1 工具注册与提示词注入

关键文件：

- `src/service/tools/tool_registry.cpp`
- `src/prompt.h`
- `src/prompt.cpp`

作用：

- 维护内置工具 schema、名称、说明、多语言描述
- 决定模型看到的工具能力清单
- 为 `calculator` / `controller` / `skill_call` / `schedule_task` / `system_engineer_proxy` / `monitor` / `MCP` 等工具提供统一注册信息

### 3.2 工具解析与执行

关键文件：

- `src/core/toolflow/tool_flow_controller.*`
- `src/service/tools/tool_executor.*`
- `src/xtool.cpp`, `src/xtool.h`

作用：

- 解析模型返回的工具调用 JSON
- 决定何时执行内置工具、何时走 MCP
- 将工具执行结果转成 `tool_response` 回写到对话流

### 3.3 典型工具路径

#### 桌面控制器 controller

- 工具执行：`src/xtool.cpp`
- UI 提示：`src/widget/controller_overlay.*`
- 远程控制通道：`src/net/controlchannel.*`
- 主窗口连接点：`src/main.cpp`

说明：

- `controller` 已支持 bbox + action + description 新 schema
- 工具层负责 bbox 到真实屏幕坐标的映射、鼠标/键盘动作执行、动作前后叠加提示
- UI 层负责屏幕提示覆盖与结果可视化

#### Skills

- 技能管理：`src/skill/skill_manager.cpp`
- 主界面联动：`src/widget/widget_skills.cpp`
- 工具注册：`src/service/tools/tool_registry.cpp`
- 工具执行分发：`src/xtool.cpp`

说明：

- `SkillManager` 负责扫描 `EVA_SKILLS`、导入 zip、启停技能、拼装技能提示块
- `skill_call` 工具负责向模型暴露 SKILL.md 内容与技能目录信息

#### MCP

- MCP 桥接：`src/xmcp.cpp`, `src/xmcp.h`, `src/xmcp_internal.cpp`
- UI 配置页：`src/expend/expend_mcp.cpp`
- 工具调用配合：`src/xtool.cpp`

说明：

- `xMcp` 管理 MCP 服务连接、工具同步、工具调用与结果回传
- 当前 MCP 配置 UI 主要位于增殖窗口

## 10. Win7 兼容相关路径

关键文件：

- `src/main.cpp`
- `src/widget/widget.cpp`
- `src/net/controlchannel.cpp`
- `tests/unit/backend/local_server_args_tests.cpp`

重点：

- 早期环境变量与运行时兼容兜底在 `main.cpp`

- 控制通道析构期对 Win7 稳定性有专门处理，位于 `src/net/controlchannel.cpp`
- 本地服务参数里有 Win7 专项测试用例
